When reviewing PRs, if a finding is worth mentioning in the review summary, it's worth a Notion task. File it in the same pass — never say "I'll file it as a follow-up" or "could fix in a future pass" without actually creating the ticket right then.

**Why:** John flagged that deferred findings get lost. "Later" never happens — the context is gone by next session and the issue silently drops off the radar.

**How to apply:** During PR review, for each finding that isn't a merge blocker: create the Notion task immediately after noting the finding. Don't wait until after the merge, don't batch them, don't leave them as notes in the review summary without a corresponding ticket.
